# 前端地图数据包

> 语言：**简体中文** | [English](map-data-package.en.md)

## 当前架构

PalOps Web 1.3.1 不再支持后端地图包导入、签名、激活、健康检查或瓦片扫描。固定地图数据是前端静态资源，随 Vue 发布包一起更新。

```text
frontend-vue/public/map/
├─ config/
├─ data/
├─ icons/
└─ tiles/
   ├─ palpagos/
   └─ world-tree/
```

构建后同样内容复制到 `src/PalOps.Web/wwwroot/map/`。

## 包含内容

- Palpagos 和 World Tree 图层配置；
- 682 个本地 WebP 栅格瓦片；
- 三种语言各 1,251 条固定 POI；
- 30 个固定分类；
- 图标、搜索别名、边界和坐标变换；
- 数据来源和许可证元数据。

## 运行时数据

后端 API 只提供：

- 在线/离线玩家；
- 公会据点；
- 自定义标记。

浏览器根据前端图层配置选择 Palpagos 或 World Tree，并执行坐标转换和边界判断。

## 首次加载

服务器实体和当前语言 POI 并行请求。图层骨架未就绪时的数据进入待提交状态，骨架完成后立即写入 GeoJSON source。底图瓦片加载不会阻塞服务器标记和默认 POI。

默认启用：服务器数据、快速传送和高塔。其他高密度分类按需创建和显示。

## 更新方式

1. 更新 `frontend-vue/public/map/data`、图标或瓦片；
2. 运行地图契约；
3. 执行 `npm run build`；
4. 发布新的前端/完整程序包。

后端不再提供“导入新地图”入口。地图更新就是前端版本更新。

## 缓存

静态资源使用内容哈希或版本化 URL 缓存。升级后如浏览器仍显示旧地图，应执行强制刷新或清理站点缓存。
